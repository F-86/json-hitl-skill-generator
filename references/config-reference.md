# 生成配置参考

## 按模式推荐的 HITL 触点数

| 模式 | Phase 数 | 推荐 CP 数 | 说明 |
|------|---------|-----------|------|
| 标准-简单 | 1-2 | 1 | 只需最终确认 |
| 标准-中等 | 3-4 | 2 | Plan 确认 + 产物审阅 |
| 标准-复杂 | 5-7 | 3 | 范围确认 + 产物审阅 + 最终交付 |
| CRUD-Read | 3-4 | 2 | 参数确认 + 结果展示（无最终确认） |
| CRUD-Create | 4-5 | 3 | 参数收集 + 预览 + 最终确认 |
| CRUD-Update | 4-5 | 3 | 参数收集 + 修改预览 + 最终确认 |
| CRUD-Delete | 4-5 | 3 | 对象选择 + 删除确认 + 最终确认（无 default） |

## 交互类型适用场景

| 类型 | 适用场景 | 注意事项 |
|------|---------|---------|
| `choice` | 范围确认、参数选择、路线决策 | options ≤ 5 个 |
| `confirm` | 是否覆盖、是否执行危险操作 | 必须能用"是/否"回答 |
| `review` | 审阅产物质量 | 必须附带 artifact_path |
| `input` | 收集项目配置、查询参数、路径 | fields ≤ 5 个 |

## CRUD 差异矩阵

| 操作 | 是否需要最终确认 | HITL 焦点 | 特殊规则 |
|------|-----------------|-----------|---------|
| **Create** (增) | ✅ 必须最终确认 | 参数收集 + 创建确认 | 操作危险，最后一步 CP 使用 `confirm` 类型 |
| **Read/Query** (查) | ❌ 不需要 | 参数收集 + 结果展示 | 参数确认后用 `choice` 让用户选择，不需要 `confirm` |
| **Update** (改) | ✅ 必须最终确认 | 参数收集 + 修改预览 + 确认 | 危险程度中等，最后一步 CP 使用 `confirm` |
| **Delete** (删) | ✅ 必须最终确认 | 对象选择 + 删除确认 | 最危险，最后一步 CP 使用 `choice`（不含全局 default），默认行为为取消 |

## 命名规则

- Skill 名称：`<task-type>-<operation>`，全小写 + 连字符
- 操作标识：`create` / `query` / `update` / `delete`
- 任务类型：由用户提供，如 `order`、`user`、`task`
- 工作区目录：`.json-hitl-gen/`
