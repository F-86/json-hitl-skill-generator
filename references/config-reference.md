# 生成配置参考

## 按模式推荐的 HITL 触点数

| 模式 | Phase 数 | 推荐 CP 数 | 说明 |
|------|---------|-----------|------|
| 标准-简单 | 1-2 | 1 | 只需最终确认 |
| 标准-中等 | 3-4 | 2 | Plan 确认 + 产物审阅 |
| 标准-复杂 | 5-7 | 3 | 范围确认 + 产物审阅 + 最终交付 |
| CRUD-Read | 3-4 | 2 | 参数确认 + 结果展示（无最终确认） |
| CRUD-Create | 4-5 | 3 | 参数收集 + 预览 + 最终确认 |
| CRUD-Update | 4-5 | 3 | 参数收集 + 修改预览 + 最终确认 |
| CRUD-Delete | 4-6 | 3-4 | 单条删除确认，或批量删除的 dry_run 预览 + 预览确认 + 最终确认（无全局 default） |

## 交互类型适用场景

| 类型 | 适用场景 | 注意事项 |
|------|---------|---------|
| `choice` | 范围确认、参数选择、路线决策 | options ≤ 5 个 |
| `confirm` | 是否覆盖、是否执行危险操作 | 必须能用"是/否"回答 |
| `review` | 审阅产物质量 | 必须附带 artifact_path |
| `input` | Create/Update 收集结构化信息 | fields ≤ 5 个；查询场景禁用 |
| `text_input` | 查询场景自由文本输入（id、名称等） | placeholder 说明多值分隔符 |
| `combobox` | 查询场景枚举字段选择 | 需提供 options_from |
| `number_range` | 查询场景数值区间 | field + label 必填 |
| `datetime_range` | 查询场景时间区间 | field + label 必填 |

## 推荐的 HITL 触发条件数

| 模式 | 最少确定性条件 | 最少可能缺失条件 | 最少歧义条件 | 说明 |
|------|-------------|----------------|------------|------|
| CRUD-Read | 0-1 | 1-2 | 0-1 | 查询参数通常较少触发 HITL |
| CRUD-Create | 1-2 | 2-4 | 0-2 | 创建参数多，不确定性高 |
| CRUD-Update | 0-1 | 2-3 | 0-2 | 定位条件 + 修改字段 |
| CRUD-Delete | 0 | 1-2 | 0-1 | 通常只需要定位条件 |

> 每个模式至少应有 1 条"可能缺失"的触发条件（否则 HITL 没有意义）。如果全部参数 LLM 都能确定，说明这个操作的复杂度太低，考虑是否需要独立的 skill。

## CRUD 差异矩阵

| 操作 | 是否需要最终确认 | HITL 焦点 | 特殊规则 |
|------|-----------------|-----------|---------|
| **Create** (增) | ✅ 必须最终确认 | 参数收集 + 创建确认 | 操作危险，最后一步 CP 使用 `confirm` 类型 |
| **Read/Query** (查) | ❌ 不需要 | 参数收集（CP-1a 内嵌 apicall）+ 结果展示 | CP-1a 的 checkpoint 内嵌 `apicall` 模板（`filters: {}`），前端提交时直接执行，不经过 LLM |
| **Update** (改) | ✅ 必须最终确认 | 参数收集 + 修改预览 + 确认 | 危险程度中等，最后一步 CP 使用 `confirm`；**批量场景**走 dry_run→CP2a→Step2，Step2 必须带 `expected_count`，Step1/Step2 条件一致 |
| **Delete** (删) | ✅ 必须最终确认 | 单条删除确认，或批量删除预览 + 确认 | 最危险；单条路径用 `choice` 最终确认（不含全局 default）；**批量路径**走 `dry_run → CP2a → CP2b`，正式执行必须带 `expected_count`，且当用户只给出单个自然语言名字、没有明确批量语义时先收集唯一标识，不要直接猜 filters |

## 命名规则

- Skill 名称：`<task-type>-<operation>`，全小写 + 连字符
- 操作标识：`create` / `query` / `update` / `delete`
- 任务类型：由用户提供，如 `order`、`user`、`task`
- 工作区目录：`.json-hitl-gen/`
